Fases del Hacking

1. Fase de reconocimiento

En esta fase, el hacker se centra en la recoleccion de información sobre el objetivo que se busca atacar. Esto puede incluir direcciones IP, sistemas operativos y configuraciones de seguridad del sistema informatico de su empresa. Estos datos se pueden obtener de manera pasiva, al investigar datos públicos como sus sitios web o redes sociales, o tambien de manera activa, lo que implica obtener los datos por parte de terceros o tecnicas de rastreo mas invasivas.

Herramientas de footprinting pasivo:

  • WHOIS:

    • Es un protocolo que permite obtener información sobre la registración de dominios, incluyendo detalles como propietario, fechas de creación y expiración, y servidores DNS asociados.

    Link de acceso: https://who.is

  • Wayback Machine:

    • Es un servicio que archiva versiones anteriores de sitios web, permitiendo acceder a contenido histórico y detectar cambios o información eliminada.

    Link de acceso: https://web.archive.org

  • Netcraft:

    • Es un sitio web que ofrece información sobre sitios web, incluyendo tecnologías utilizadas, ubicación de servidores y historial de hosting.

    Link de acceso: https://www.netcraft.com

Herramientas de footprinting activo:

  • Consola de comandos

    • Interfaz de línea de comandos que permite ejecutar diversos comandos para interactuar con el sistema operativo y realizar tareas de red.

    Comandos de ejemplo: "ping, tracert, nslookup"

  • Recon-NG

    • Framework de reconocimiento que facilita la recopilación de información mediante módulos automatizados, permitiendo realizar tareas de OSINT de manera eficiente.

    Link de descarga: https://github.com/lanmaster53/recon-ng

  • Nmap

    • Herramienta de escaneo de redes que permite descubrir hosts y servicios en una red, identificar puertos abiertos y detectar vulnerabilidades.

    Link de descarga: https://nmap.org/download

    Comandos de ejemplo: "nmap prismati.cl"


Medida de proteccion/mitigacion:

  • Limitar la exposición de información y proteger contra ingeniería social.
  • Bloqueo de IPs sospechosas asociadas a actividades de reconocimiento.
  • Monitoreo de tráfico entrante con sistemas de detección de intrusos (IDS).
  • Capacitación en ingeniería social para el personal.

2. Fase de escaneo

En esta fase se profundizan los conocimientos sobre el objetivo. Esto implica que, mediante el escaneo, el hacker se encarga de identificar los dispositivos, puertos, programas, servicios que se ejecutan. Ademas, investiga las vulnerabilidades.

Herramientas de scanning:

  • Nmap:

    • Es una herramienta de código abierto utilizada para el descubrimiento de redes y auditoría de seguridad. Permite identificar hosts y servicios en una red, detectar puertos abiertos y determinar el sistema operativo de los dispositivos conectados.

    Link de descarga: https://nmap.org/download

    Comandos basicos:

    • nmap prismati.cl - Escanea los 1,000 puertos TCP más comunes en el host prismati.cl
    • nmap -p 20-80 prismati.cl - Escanea los puertos TCP desde el 20 hasta el 80 en el host prismati.cl
    • nmap -sV prismati.cl - Detecta las versiones de los servicios que se ejecutan en los puertos abiertos del host prismati.cl
    • nmap -O prismati.cl - Intenta identificar el sistema operativo del host prismati.cl mediante análisis de la pila TCP/IP
    • nmap -F prismati.cl - Realiza un escaneo rápido de los puertos más comunes en el host prismati.cl

  • OpenVAS:

    • Es un escáner de vulnerabilidades del sistema que verifica los puertos visibles, los servicios a los que puede acceder en busca de vulnerabilidades conocidas y amenazas web de alto nivel

    Link de descarga: https://www.openvas.org

  • Nikto:

    • Es un escáner de vulnerabilidades de aplicaciones web de código abierto que realiza pruebas exhaustivas en servidores web para identificar archivos peligrosos, configuraciones inseguras y otros problemas de seguridad.

    Link de descarga: https://github.com/sullo/nikto


Medida de proteccion/mitigacion:

  • Controlar el acceso a los puertos, monitorear la actividad sospechosa y minimizar la exposición de información técnica.
  • Configuración de firewalls estrictos para bloquear puertos innecesarios.
  • Implementación de IDS/IPS para detectar y bloquear escaneos.
  • Realización de pruebas internas de penetración para detectar vulnerabilidades.
  • Ocultación de versiones de servicios y cabeceras.

3. Fase de acceso

En esta etapa, con los datos ya obtenidos, el hacker procede a simular ataques a los sistemas en un entorno controlado, para ingresar en ellos las vulnerabilidades identificadas. Aqui es donde comenzará a poner a prueba la capacidad defensiva de los sistemas ante ataques como inyecciones SQL, ataques XSS, etc.

Tipos de ataque:

  • Inyección SQL:

    • Es un tipo de ciberataque encubierto en el cual un hacker inserta código propio en un sitio web con el fin de quebrantar las medidas de seguridad y acceder a datos protegidos. Una vez dentro, puede controlar la base de datos del sitio web y secuestrar la información de los usuarios.
    Existen distintos tipos de inyeccion SQL, estos son:

    • Inyección SQL en banda:

      • La inyección SQL en banda es un tipo de ataque en el que un atacante interactúa directamente con la base de datos a través de una aplicación vulnerable, aprovechando respuestas inmediatas del servidor para ejecutar consultas maliciosas.

      Métodos comunes:
      - Ataque basado en la Unión: Utiliza la cláusula UNION para combinar resultados de consultas maliciosas con las consultas legítimas.
      - Ataque basado en errores: Aprovecha mensajes de error generados por el servidor para obtener información sobre la estructura de la base de datos.

    • Inyección SQL Inferencial (Blind SQL Injection):

      • La inyección SQL inferencial (o inyeccion SQL ciega) es un tipo de ataque en el que el atacante no recibe resultados directos de sus consultas, sino que deduce la información basándose en cómo responde el servidor a ciertas condiciones.

      Métodos comunes:
      - Ataque Booleano: En una inyección SQL booleana, un atacante evalúa qué partes de la entrada de un usuario son vulnerables a las inyecciones SQL probando dos versiones diferentes de una cláusula booleana a través de la entrada
      - Ataque basado en el tiempo: Un atacante utiliza una función predefinida basada en el tiempo del sistema de administración de la base de datos que es utilizada por la aplicación. Por ejemplo, en MySQL, la función sleep() le indica a la base de datos que espere un cierto número de segundos.

    • Inyección SQL fuera de banda:

      • La inyección SQL fuera de banda es un tipo de ataque en el que el atacante utiliza canales secundarios para extraer información o interactuar con la base de datos, sin depender directamente de la respuesta de la aplicación web. En MySQL, las funciones LOAD_FILE() y INTO OUTFILE pueden ser usadas para solicitar a MySQL que transmita los datos a una fuente externa.

  • Ataque XSS:

    • Es un tipo de ataque en aplicaciones web que permite a un atacante inyectar scripts maliciosos en páginas web vistas por otros usuarios. Estos scripts suelen ejecutarse en el navegador del cliente y pueden robar datos sensibles o manipular la interfaz.

    • XSS Reflejado (Reflected XSS):

      • Este tipo de XSS ocurre cuando el script malicioso se envía como parte de una solicitud (por ejemplo, a través de un enlace o formulario) y el servidor lo refleja directamente en la respuesta sin validarlo. El script se ejecuta en el navegador de la víctima cuando interactúa con el enlace o el formulario comprometido.

    • XSS Almacenado (Stored XSS):

      • El script malicioso se guarda de forma permanente en el servidor, generalmente en bases de datos o publicaciones de usuarios (como comentarios o perfiles). Cada vez que otros usuarios acceden al contenido afectado, el script se ejecuta automáticamente en sus navegadores.

  • Desbordamiento de Buffer:

    • Un desbordamiento del buffer es una situación en la que un programa en ejecución intenta escribir datos fuera del buffer de memoria que no está destinado a almacenar estos datos. Cuando esto sucede, estamos hablando de un desbordamiento del buffer.

    Las vulnerabilidades de desbordamiento del buffer se producen en todo tipo de software, desde sistemas operativos hasta aplicaciones cliente / servidor y software de escritorio. Esto sucede a menudo debido a una mala programación y la falta de validación de entrada o deficiente en el lado de la aplicación.

    • Consecuencias

      • Cuando se produce un desbordamiento del buffer de memoria y los datos se escriben fuera del buffer, el programa en ejecución puede volverse inestable, bloquearse o devolver información corrupta.

      Las partes sobrescritas de la memoria pueden haber contenido otros datos importantes para la aplicación en ejecución que ahora se sobrescribe y ya no está disponible para el programa. Los desbordamientos del buffer pueden incluso ejecutar otros programas o comandos (maliciosos) y provocar la ejecución de código arbitrario.

Medidas de proteccion contra ataques:

  • Inyección SQL
    • Rechazar caracteres especiales como ', ", --, ; o /* que puedan usarse para manipular consultas SQL.
    • Implementar sentencias parametrizadas en lugar de concatenar entradas directamente en las consultas.
    • No exponer detalles técnicos en mensajes de error, ya que pueden proporcionar información útil para los atacantes.
  • Ataque XSS
    • Usar funciones que conviertan caracteres peligrosos (<, >, ', ") en entidades seguras. Por ejemplo la funcion de htmlspecialchars() puede convertir caracteres especiales en PHP
    • Validar todos los datos que ingresen los usuarios, rechazando cualquier código malicioso.
  • Desbordamiento de Buffer
    • Evitar que ocurran condiciones de desbordamiento del buffer en el código.
      Por ejemplo, cuando se espera un máximo de 8 bytes como datos de entrada, la cantidad de datos que se puede escribir en el buffer se limitará a 8 bytes en cualquier momento.

4. Fase de mantenimiento de acceso

Una vez el hacker ya haya vulnerado el sistema, el trabajo consiste en mantener ese acceso el mayor tiempo posible, con el fin de realizar multiples ataques y asi obtener hackear el sistema. Para seguir dentro del objetivo infectado, el hacker puede hacer uso de malware para controlar los sistemas y equipos sin ser detectado.

Tecnicas para mantenimiento de acceso:

  • Backdoors (Puertas traseras):

    • Instalación de programas o scripts que permiten acceso remoto al sistema sin pasar por controles de autenticación.

  • Escalación de privilegios:

    • Obtener privilegios administrativos en el sistema para evitar restricciones de acceso. Por ejemplo exploits específicos para vulnerabilidades del sistema operativo.

  • Abuso de cuentas legítimas:

    • Capturar credenciales de usuario y utilizarlas para acceder al sistema de manera legítima, dificultando su detección.

Politicas de seguridad:

  • Implementación de autenticación en dos pasos:

    • Requiere que los usuarios verifiquen su identidad mediante al menos dos factores: algo que saben (contraseña), algo que tienen (token o dispositivo móvil) o algo que son (biometría). Esto dificulta que un atacante use credenciales robadas.

  • Gestión estricta de privilegios mínimos:

    • Asegura que cada usuario solo tenga los permisos necesarios para realizar su trabajo, evitando que cuentas con acceso innecesario puedan ser explotadas. Esto incluye la segmentación de privilegios en sistemas críticos.

  • Política de rotación y complejidad de contraseñas:

    • Requiere que las contraseñas sean robustas (longitud mínima, combinación de caracteres) y se cambien regularmente. Además, el uso de un gestor de contraseñas reduce el riesgo de reutilización de credenciales.

5. Fase de borrado de huellas

Finalmente, la tarea final del hacker consiste en eliminar cualquier rastro de su presencia en el objetivo que se ha atacado (computadoras, sitios web, redes). Para esto puede ocultar su direccion IP, eliminar archivos de registro o utilizar conexiones cifradas.

Herramientas:

  • Visor de eventos de Windows:

    • El Visor de eventos es una herramienta integrada en Windows que permite visualizar y gestionar los registros (logs) del sistema operativo. Estos registros documentan actividades clave del sistema, aplicaciones, y eventos de seguridad.

    Caracteristicas principales:

    • Almacena eventos categorizados por tipo: sistema, seguridad, aplicaciones, entre otros.
    • Permite filtrar y buscar eventos específicos.
    • Proporciona detalles como fecha, hora, usuario, origen del evento y nivel de gravedad (información, advertencia, error).

    Información que almacena:

    • Eventos de inicio de sesión
    • Errores de aplicaciones
    • Cambios en la configuración del sistema
  • Logs de Linux (archivos de registro):

    • Los sistemas Linux almacenan registros en archivos de texto ubicados generalmente en el directorio /var/log/. Estos logs documentan actividades del sistema, seguridad, servicios y aplicaciones.

    Caracteristicas principales:

    • Son accesibles y editables mediante un editor de texto o herramientas como cat, tail o less.
    • Los logs son gestionados por syslog o journald, dependiendo de la distribución.
    • Ofrecen una estructura cronológica y detallada de los eventos.

    Información que almacena:

    • /var/log/auth.log: eventos relacionados con la autenticación (inicios de sesión exitosos y fallidos).
    • /var/log/syslog o /var/log/messages: registros generales del sistema.
    • /var/log/kern.log: mensajes del núcleo (kernel).
    • /var/log/httpd/: logs de servidores web Apache.

Prevención del borrado de logs

  • Restricción de acceso:

    • Asegurar que solo usuarios privilegiados (por ejemplo, administradores o root) tengan permisos para leer o modificar los logs.

  • Centralización de registros:

    • Enviar los logs a un servidor centralizado de registro (como SIEM, Splunk, o Graylog). Esto asegura que los eventos se guarden fuera del sistema comprometido, dificultando su eliminación.

  • Configuración de inmutabilidad:

    • En Linux, usar el comando chattr +i en los archivos de log para marcarlos como inmutables, evitando que puedan ser modificados o borrados incluso por el usuario root.

  • Monitoreo y alertas:

    • Configurar herramientas como Tripwire o OSSEC para detectar cambios o intentos de eliminación de los archivos de registro.

Navegación anónima

  • Uso de la red Tor:

    • Permite navegar de manera anónima redirigiendo el tráfico a través de nodos intermedios cifrados, ademas de ofrecer acceso a servicios ocultos en la dark web.

  • VPN:

    • Oculta la dirección IP real al enrutar el tráfico a través de un servidor remoto, proporcionando cifrado para evitar que terceros intercepten la conexión.

  • Uso de sistemas operativos enfocados en anonimato:

    • Tails es un sistema operativo basado en Linux diseñado para preservar la privacidad y el anonimato.

  • Navegadores enfocados en privacidad:

    • Navegadores como Brave o Tor Browser bloquean rastreadores y eliminan cookies automáticamente.
